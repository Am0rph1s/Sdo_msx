# Cartridge Build Process — Nau DX

## Overview

The final ROM (`nau_dx_intro.rom`) is a 64KB ASCII16 cartridge containing:
- **Segment 0** (bank 0, 0x4000–0x7FFF): Intro code (~2KB, fixed at boot)
- **Segment 1** (bank 1, 0x8000–0xBFFF): SC2 intro image (Pletter-compressed, default visible)
- **Segment 2** (bank 2, switched to 0x4000): Game first 16KB
- **Segment 3** (bank 3, switched to 0x8000): Game second ~10KB

Intro→game transition uses a **RAM trampoline** at 0xC000 (copied from intro code) that switches both banks then jumps to 0x4000.

## Build Steps (in order)

### 1. Compress intro image

```
tools\utils\pletter.exe portada2.sc2 intro_sc2.plet
```

Input: SC2 file (BLOAD header + VRAM data, 14336–16384 bytes)
Output: Pletter-compressed data (~3500 bytes)

### 2. Build the game binary

1. Copy `project_config_game.js` → `project_config.js`
2. Run: `build.bat`
3. Output: `out\nau_dx.rom` (32KB ROM_32K)
4. Split into two 16KB parts:
   - `game_p1.bin` = first 16384 bytes
   - `game_p2.bin` = remaining bytes

### 3. Build the intro + cartridge

1. Copy `project_config_intro.js` → `project_config.js`
2. Run: `build.bat`
3. Output: `out\nau_dx_intro.rom` (64KB ASCII16)

### 4. Deploy

Copy `out\nau_dx_intro.rom` to emulator or flash cart.

## ROM Layout Details

```
0x0000–0x3FFF  Segment 0: intro code + CRT0 / header
0x4000–0x7FFF  Bank 0 (fixed): intro code (~2KB), rest padded
0x8000–0xBFFF  Bank 1 (default): intro_sc2.plet
               (switched to segment 3 at runtime)
0xC000–0xFFFF  RAM: trampoline code, game stack/vars
```

## Bank Switching (ASCII16)

- `LD (0x6000), A` → switch segment `A` into bank 0 (0x4000–0x7FFF)
- `LD (0x77FF), A` → switch segment `A` into bank 1 (0x8000–0xBFFF)
- Default at boot: segment 0 → bank 0, segment 1 → bank 1

## RAM Trampoline (at 0xC000)

```asm
LD A, 2         ; segment 2 = game_p1.bin
LD (0x6000), A  ; switch bank 0
LD A, 3         ; segment 3 = game_p2.bin
LD (0x77FF), A  ; switch bank 1
JP 0x4014       ; jump to game entry
```

## Important Rules

- **Never switch bank 0 while running code from it** — use the RAM trampoline
- The game source (`nau_dx.c`) must compile as ROM_32K (no mapper code)
- When modifying game code, rebuild game first, then the intro cartridge
- When changing the intro image, recompress and rebuild only the intro
- The trampoline is in `nau_dx_intro.c`: update segment numbers there if layout changes
- Game entry point must be at address 0x4014 (after CRT0 init)
